{# ── Journey 1: New user ── #}

Journey 1: New user registers and creates a service

When I need to integrate my service with One Login, I want to create an admin account and register my service, so that I can start configuring my integration.

  1. 1 Start page — landing page with "Create an account" button
  2. 2 Create an account — name, email, mobile, password
  3. 3 Your services — empty state, click "Add a new service"
  4. 4 Create a service — overview of what you'll need
  5. 5 Enter service name
  6. 6 Select authentication method — JWKS, static key, or client secret
  7. 7 Add redirect URLs
  8. 8 Select scopes — choose user information needed
  9. 9 Check your answers — review and create service
Start this journey
{# ── Journey 2: Returning user ── #}

Journey 2: Returning user signs in and manages a service

When I already have an account, I want to sign in and update my service configuration, so that I can keep my integration up to date.

  1. 1 Start page
  2. 2 Sign in — email and password
  3. 3 Enter security code — SMS verification
  4. 4 Your services — select a service to manage
  5. 5 Client configuration — integration and production tabs

From client configuration, you can change individual fields:

  • Text/textarea fields (name, public key, contacts, etc.)
  • Radio selections (auth method, key type, boolean toggles)
  • Checkbox selections (scopes, claims, levels of confidence)
  • URL lists with add/remove (redirect URIs, logout URIs)
Start this journey
{# ── Journey 3: Go live ── #}

Journey 3: Make a service live

When my integration is ready, I want to complete the go-live checklist and submit a request, so that my service can move to production.

Note: These pages are accessed via the sidebar within a service (e.g. /services/:uid/make-live). The links below use backward-compatible routes that redirect to the first service.

  1. 1 Go-live checklist — task list with completion status
  2. 2 Confirm integration is complete
  3. 3 Update production redirect URLs
  4. 4 Configure production scopes
  5. 5 Confirm 2+ admins on the service
  6. 6 Accept memorandum of understanding
  7. 7 Request sent — confirmation panel
Start this journey
{# ── Journey 4: Team & account management ── #}

Journey 4: Manage team and account

When I need to collaborate with colleagues, I want to invite team members and manage account settings, so that the right people have access.

  1. 1 Your account — personal details and sign out
  2. 2 Team members — list of current members
  3. 3 Invite a team member — email and permission level
  4. 4 Settings — service name, client ID, contacts
  5. 5 Audit log — history of configuration changes
