Features and roadmap

One place for government services to independently set up, manage and maintain their One Login integrations.

This roadmap shows what you can do now and what we're planning. It signals intention so some things may change.

You can contact us if you have questions, want to get involved in user research, or have suggestions.

Last updated: March 2026

Next review due: June 2026

What you can do now

Set up your integration

  • Create a new integration environment configuration
  • Generate and manage your API credentials
  • Configure redirect URIs and basic client settings

Manage your integration

  • View your integration environment settings
  • Update configuration values as your service develops
  • Access your client ID and authentication details

Things we're working on

We're working with government service teams to improve the admin tool. Here's what's coming.

Multi-user access

  • Add team members to manage your integration
  • Keep your service running when people move on
  • Set permissions so the right people can make changes

Better configuration management

  • See all your configuration in one place, including settings not currently visible
  • View your production configuration to compare it with your integration environment

Transparency and audit

  • See a history of changes to your configuration
  • Know who changed what and when

Things we'll do next

  • Make it easier to update your configuration without raising a support ticket
  • Improve the security and reliability of how configuration changes are made
  • See how your service is using One Login, including authentication volumes and error rates
  • Get notified when you need to make a change to your configuration

Things we'll do later

  • Let services update their production configuration directly
  • Support more One Login configuration options, including GOV.UK Wallet
  • Reduce the need for manual support during setup and maintenance
  • Let you manage services via API as part of your deployment pipeline
  • Guide you through moving from integration to production within the tool
  • Help you adopt features that improve your service's reliability and security
  • Detect when rate limits are being hit so we can proactively offer support
  • Identify dormant services so we can help teams keep their integrations healthy
